# Recurso de segurança do Windows 11 volta a exigir formatação para ser ativado
Um dos mecanismos de proteção do Windows 11, o Controle inteligente de aplicativos, continuará exigindo a reinstalação completa do sistema caso seja desativado. A possibilidade de reativar o recurso sem formatar o computador foi adiada novamente, conforme indica uma página de suporte da Microsoft.

## Resumo em 3 bullets
- Controle inteligente de aplicativos verifica se programas são maliciosos antes da primeira execução
- Uma vez desativado, recurso exige reinstalação completa do Windows para ser reativado
- Microsoft adiou flexibilização que permitiria ativar/desativar sem formatação

## Contexto
O Controle inteligente de aplicativos (Smart App Control) é um mecanismo de segurança pró-ativo do Windows 11 que verifica se programas são maliciosos antes de serem executados pela primeira vez. O sistema analisa os arquivos do app e, caso nenhuma ameaça seja detectada, libera a execução sem interrupções. No entanto, uma limitação relevante é que uma vez desativado, não é possível ativá-lo novamente sem formatar completamente o Windows, pois o sistema passa a ser considerado potencialmente inseguro após o desligamento da função.

## Insights e implicacoes
A Microsoft havia anunciado em janeiro que tornaria o recurso mais flexível, permitindo ativar e desativar o Controle inteligente de aplicativos a qualquer momento, sem a necessidade de reinstalar o sistema operacional. No entanto, essa mudança aparentemente foi adiada, conforme indicado em uma página de suporte relacionada à atualização, onde foi incluído o aviso de que o recurso foi removido da documentação e está planejado para um lançamento futuro. A empresa não detalhou os motivos do adiamento nem apresentou uma nova previsão para disponibilizar a função ao público geral.

- Monitore atualizações da Microsoft sobre a flexibilização do recurso
