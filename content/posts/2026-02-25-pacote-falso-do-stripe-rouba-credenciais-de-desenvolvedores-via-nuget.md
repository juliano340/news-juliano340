# Pacote falso do Stripe rouba credenciais de desenvolvedores via NuGet
Pesquisadores identificaram pacote malicioso no NuGet que se passa pela biblioteca oficial do Stripe para roubar tokens de API.

## Resumo em 3 bullets
- Pacote falso StripeApi. Net imita biblioteca oficial do Stripe no NuGet
- Criminosos usam typosquatting para enganar desenvolvedores
- Código malicioso captura tokens de API e envia para criminosos via Supabase

## Contexto
Desenvolvedores usam repositórios como NuGet para instalar pacotes de código prontos. A abertura desses repositórios permite que criminosos publiquem versões falsas de bibliotecas populares. O pacote malicioso StripeApi. Net imitava perfeitamente o original, incluindo documentação e ícone, mas continha código para roubar credenciais.

## Insights e implicacoes
O ataque mostra que contagem de downloads não é garantia de segurança. Criminosos inflaram artificialmente o número de downloads para parecer popularidade orgânica. O uso de serviços legítimos como Supabase para exfiltração dificulta detecção. Ataques à cadeia de suprimentos afetam múltiplas vítimas através de uma única biblioteca comprometida.

## O que fazer agora
- Verificar nome exato do pacote antes de instalar
- Checar perfil do publicador por logos e verificação
- Usar ferramentas de análise para escanear pacotes
- Manter inventário atualizado de dependências
- Desconfiar de pacotes novos com downloads altos

## O que vale acompanhar
- Ataques usando typosquatting em repositórios de código
- Uso de serviços legítimos para exfiltração de dados
- Supply chain attacks afetando bibliotecas populares
- Criminosos inflando artificialmente contagem de downloads
- Comprometimento de pacotes legítimos em npm e NuGet
