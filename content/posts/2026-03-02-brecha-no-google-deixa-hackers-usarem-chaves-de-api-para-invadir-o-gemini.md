# Brecha no Google deixa hackers usarem chaves de API para invadir o Gemini
Chaves de API da Google para serviços como o Maps, embutidas em códigos do lado do cliente, podem ser exploradas para autenticação na IA Gemini na obtenção de dados privados.

## Resumo em 3 bullets
- Chaves de API da Google Cloud expostas online podem autenticar no Gemini
- Mais de 2.800 chaves foram encontradas em sites populares em 2025
- Problema se originou com introdução do Gemini como assistente

## Contexto
A descoberta foi feita pelos pesquisadores da TruffleSecurity ao monitorar páginas da internet de diversas organizações. Antes do Gemini, chaves de API do Google Cloud não eram consideradas dados sensíveis e podiam ser expostas online sem risco. O problema surgiu quando o Gemini passou a usar essas mesmas chaves como credenciais de autenticação.

## Insights e implicacoes
A mudança repentina na segurança das chaves de API criou uma vulnerabilidade crítica, pois desenvolvedores não foram devidamente alertados sobre o novo risco. A exploração dessas chaves pode resultar em custos significativos e acesso a dados privados.

## O que fazer agora
- Verificar se a API de linguagem generativa está ativada em projetos
- Auditar todas as chaves de API presentes no ambiente
- Implementar medidas para detectar e bloquear chaves vazadas

## O que vale acompanhar
- Comunicados oficiais do Google sobre a vulnerabilidade
- Atualizações do AI Studio para tornar chaves exclusivas ao Gemini
- Ferramentas de detecção de chaves de API expostas
